## Authentication

All endpoints require mTLS authentication with optional API key headers:

<CodeGroup>

```bash curl
curl -X POST "https://sandbox.api.visa.com/visaaliasdirectory/v1/resolve" \
  --cert client.pem \
  --key client.key \
  --cacert ca.pem \
  -H "Content-Type: application/json" \
  -d '{"alias": "user@example.com", "aliasType": "EMAIL"}'
```

```typescript TypeScript
import { SecureHttpClient } from 'visa-direct-sdk';

const client = new SecureHttpClient({
  baseUrl: 'https://sandbox.api.visa.com',
  certPath: './certs/client.pem',
  keyPath: './certs/client.key',
  caPath: './certs/ca.pem'
});
```

```python Python
from visa_direct_sdk.transport.secure_http_client import SecureHttpClient

client = SecureHttpClient(
    base_url='https://sandbox.api.visa.com',
    cert_path='./certs/client.pem',
    key_path='./certs/client.key',
    ca_path='./certs/ca.pem'
)
```

</CodeGroup>

## Production Considerations

<Warning>
  This API specification is optimized for developer experience. For production implementations, refer to the official Visa Developer documentation for complete payload requirements and compliance guidelines.
</Warning>

- **mTLS Certificates**: Required for all production environments
- **JWE Encryption**: Mandatory for sensitive data endpoints
- **Idempotency**: Use unique keys for retry scenarios
- **Rate Limits**: Implement proper throttling and retry logic
- **Monitoring**: Set up comprehensive logging and alerting
